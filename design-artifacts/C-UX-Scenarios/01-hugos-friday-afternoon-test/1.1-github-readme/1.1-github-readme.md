## Overview

**Page Purpose:** Trigger a moment of recognition via mirror-code and pass Hugo's 90-second trust filter.

**User Situation:** Hugo is at his lab desk on Friday at 3:30 PM, exhausted by a failing custom pipeline and looking for a credible way to adapt a World Model without rewriting infrastructure.

**Main User Goal:** Evaluate in less than 90 seconds whether PhysLink is reliable, understandable, and worth trying immediately in Colab.

**Success Criteria:** Hugo recognizes the pipeline problem, trusts the repo enough to continue, and clicks `Open in Colab`.

**Entry Points:**
- Research Slack link.
- Twitter/X research post.
- arXiv or paper referral.
- Recommendation from a lab colleague.

**Exit Points:**
- Primary: `Open in Colab`.
- Secondary: `Read the paper`.
- Tertiary: deep documentation links for debugging, domain science, lab adoption, changelog, contribution.

## Layout Structure

The README must front-load trust and code. The first viewport should show the project identity, trust badges, the problem hook, the foundational code example, and the `Open in Colab` path without forcing Hugo into long documentation reading.

## Spacing

**Scale:** WDS local tokens: `zero`, `sm`, `md`, `lg`, `xl`, `2xl`, `3xl`.

| Property | Token |
|----------|-------|
| Page padding (horizontal) | GitHub README default content gutter |
| Section gap | `lg` default |
| Element gap within sections | `md` default |
| Component gap within groups | `sm` default |

#### <-> `readme-v-space-md` - repository header and hero remain close

Repository metadata and the project message must read as one coherent trust surface.

#### <-> `readme-v-space-lg` - hook breathes before proof by code

The code example is the proof moment and needs visual separation from the headline.

#### <-> `readme-v-space-sm` - result belongs directly to code

The terminal output should feel like the direct consequence of the code block.

#### <-> `readme-v-space-md-action` - CTA follows proof

The action bar should appear immediately after the result proof.

#### <-> `readme-v-space-xl` - fast path to practical documentation

Installation and quickstart are secondary to `Open in Colab`, so they need a clear boundary.

#### <-> `readme-v-space-lg-trust` - advanced links remain related

Deep trust links support skeptical review without dominating the fast path.

#### <-> `readme-v-space-xl-footer` - exploration separates from contribution

Contribution links sit after the user has enough context to decide whether the repo is worth engaging with.

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | README loaded normally. | Title, hook, badges, code example, result proof, and Colab CTA are visible in the early README flow. | Read, copy, open Colab, read paper. |
| Badge unavailable | CI, arXiv, or license badge does not load. | Fallback text badge appears without breaking the main line. | Continue reading. |
| Clipboard success | A copy action succeeds. | Button label changes to `Copied` for 2 seconds. | Continue; copy again after reset. |
| Clipboard error | Clipboard API is unavailable or blocked. | Inline fallback: `Select and copy manually`. | Select code manually. |
| External target unavailable | Colab or arXiv target is unreachable. | Link remains visible; browser/GitHub displays target error. | Return to README. |
| Mobile / narrow viewport | README is viewed on a narrow screen. | Code block scrolls horizontally; CTAs stack; text stays inside content bounds. | Scroll and tap links. |
| Repository archived | Project is paused, replaced, or no longer maintained. | Banner at top states status and points to successor/fork if any. | Read status or navigate to successor. |

## Validation & Errors

This README has no user input form, so there is no field-level runtime validation. Validation is documentation/release validation plus clipboard fallback behavior.

| Field / Target | Rule | Timing | Error / Recovery |
|----------------|------|--------|------------------|
| Colab notebook URL | Must point to a public, preferably versioned notebook. | Pre-release / CI docs check. | `Colab notebook target is unavailable or unversioned.` |
| arXiv paper URL | Must point to a public arXiv page. | Pre-release / CI docs check. | `Paper link is unavailable.` |
| Code example | Must be executable or explicitly labeled as pseudo-code. | Docs test / lint. | `README code example does not match current API.` |
| Install command | Must match the published package name or approved temporary name. | Release check. | `Install command does not match published package name.` |
| Benchmark claim | Must have a source or be qualified as an example estimate. | Publication review. | `Time-saved claim needs evidence or softer wording.` |
| Clipboard action | Must provide manual fallback. | Runtime interaction. | `Select and copy manually.` |
