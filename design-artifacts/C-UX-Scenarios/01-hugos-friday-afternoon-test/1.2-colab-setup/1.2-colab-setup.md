## Overview

**Page Purpose:** Get Hugo from a public Colab notebook to a verified "ready to configure" environment with minimal uncertainty.

**User Situation:** Hugo clicked `Open in Colab` from the GitHub README. He is still skeptical and wants proof that setup will not consume the rest of his afternoon.

**Main User Goal:** Install PhysLink and run `physlink doctor` in under 30 seconds of notebook work.

**Success Criteria:** Hugo sees a clear `GO` diagnostic result and proceeds to the Universal Space API configuration cell.

**Entry Points:**
- `Open in Colab` button from `1.1-github-readme`.
- Direct Colab notebook URL shared by a colleague.
- Documentation quickstart link.

**Exit Points:**
- Primary: continue to `1.3-colab-configuration`.
- Secondary: open troubleshooting if the diagnostic returns `NO-GO`.
- Tertiary: return to GitHub README or docs.

## Layout Structure

The notebook should feel like a narrow operational checklist, not a tutorial essay. Hugo should run two cells, see a structured environment report, and move on.

```text
+----------------------------------------------------------+
| Colab Notebook Header                                    |
| notebook title, runtime badge, GitHub source link         |
+----------------------------------------------------------+
| Setup Promise                                            |
| time promise + what will be checked                      |
+----------------------------------------------------------+
| Runtime Checklist                                        |
| GPU/runtime requirement and quick manual fallback         |
+----------------------------------------------------------+
| Install Cell                                             |
| pip install command + expected install output             |
+----------------------------------------------------------+
| Doctor Cell                                              |
| physlink doctor command                                  |
+----------------------------------------------------------+
| Environment Report                                       |
| Python, PyTorch, CUDA, VRAM, session estimate, verdict    |
+----------------------------------------------------------+
| Next Step Gate                                           |
| continue to configuration or open troubleshooting         |
+----------------------------------------------------------+
```

#### <-> `colab-setup-v-space-sm` - header and setup promise stay connected

The notebook title and setup promise should read as a single entry surface.

#### <-> `colab-setup-v-space-md` - promise to runtime checklist

The checklist is part of setup, but it needs enough separation to become scannable.

#### <-> `colab-setup-v-space-sm-install` - install cell and output stay coupled

The install command and expected output belong together.

#### <-> `colab-setup-v-space-sm-doctor` - doctor cell and report stay coupled

The diagnostic report must feel like the immediate result of running `physlink doctor`.

#### <-> `colab-setup-v-space-lg-next` - diagnostic result to next gate

The next action appears after Hugo has processed the verdict.

## Page Interactions

| Object ID | Interaction |
|-----------|-------------|
| `colab-setup-header-source-link` | Opens the versioned notebook source in GitHub in a new tab. |
| `colab-setup-runtime-settings-link` | Shows or links to instructions for selecting a GPU runtime in Colab. |
| `colab-setup-install-code-cell` | User runs the cell. Notebook installs PhysLink and streams package output. |
| `colab-setup-doctor-code-cell` | User runs the cell. `physlink.doctor()` scans Python, PyTorch, CUDA, VRAM, and session estimate. |
| `colab-setup-next-configuration-link` | Moves to `1.3-colab-configuration`; enabled or visually emphasized only after `GO`. |
| `colab-setup-next-troubleshooting-link` | Opens troubleshooting guidance for failed or warning checks. |

**Global Interaction Rules:**
- Each code cell should be runnable top-to-bottom without hidden prerequisites.
- Diagnostic output should appear within 15 seconds for the environment scan itself.
- Warnings may allow continuation only when they do not threaten the Friday afternoon path.
- Failures must name the specific corrective action before the next attempt.

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Notebook opens before any cells run. | Header, promise, checklist, install cell, and doctor cell are visible. | Run install cell. |
| Runtime disconnected | Colab runtime is not connected. | Native Colab disconnect state plus setup note: `Connect runtime before installing.` | Connect runtime. |
| Installing | Install cell is running. | Cell spinner and package logs. | Wait; do not run doctor yet. |
| Install success | `pip install` completes. | Expected output includes `Successfully installed physlink`. | Run doctor cell. |
| Install error | Package install fails. | Error output plus troubleshooting link. | Retry install or open troubleshooting. |
| Doctor running | Diagnostic cell is running. | Status text: `Checking environment...`. | Wait. |
| Doctor GO | Critical checks pass. | Green/positive verdict: `GO: ready for Universal Space API configuration.` | Continue to configuration. |
| Doctor warning | Non-critical check is degraded. | Yellow warning row with concrete impact. | Continue if acceptable or fix runtime. |
| Doctor NO-GO | Critical check fails. | Error callout naming the failed requirement. | Fix runtime, rerun doctor. |
| CPU-only runtime | No CUDA GPU is available. | `NO-GO` or strong warning depending on demo mode; explains how to select GPU runtime. | Change runtime type. |
| Low VRAM | GPU memory is below preferred threshold. | Warning or `NO-GO` based on minimum requirement. | Switch runtime or reduce demo. |

## Validation & Errors

| Field / Target | Rule | Timing | Error / Recovery |
|----------------|------|--------|------------------|
| Notebook source URL | Must point to a public versioned notebook or release tag. | Pre-release docs check. | `Quickstart notebook target is unavailable or unversioned.` |
| Install command | Must install the same package/API used by notebook imports. | Docs test / CI. | `Install command does not match notebook import path.` |
| Python version | Must satisfy PhysLink runtime requirement, expected Python 3.10+. | `physlink doctor`. | `Python version unsupported. Use Python 3.10 or newer.` |
| PyTorch availability | Must import successfully. | `physlink doctor`. | `PyTorch is unavailable. Reinstall runtime dependencies.` |
| CUDA availability | Required for the fast Colab path. | `physlink doctor`. | `CUDA runtime not detected. Select a GPU runtime in Colab.` |
| GPU memory | Preferred minimum 8 GB VRAM for the Friday afternoon demo. | `physlink doctor`. | `GPU memory is below the demo target. Use T4 or reduce the demo profile.` |
| Diagnostic runtime | Environment scan should complete in under 15 seconds. | `physlink doctor`. | `Environment check timed out. Restart runtime and rerun doctor.` |
| Continue gate | Configuration section should be emphasized only after a valid GO state. | Notebook state / user flow. | `Run physlink doctor before continuing.` |

## Conditional Sections

| Condition | Include |
|-----------|---------|
| Public notebook | Make the first two cells self-contained and safe to run. |
| Needs runtime data | `physlink doctor` must report Python, PyTorch, CUDA, VRAM, and session estimate. |
| Error recovery | Every `NO-GO` row needs a direct corrective action. |
| Accessibility | Markdown headings must form a sane hierarchy; diagnostic output must not rely on color alone. |
| Open questions | Confirm final notebook URL, package name, and whether CPU-only demo mode is allowed. |
