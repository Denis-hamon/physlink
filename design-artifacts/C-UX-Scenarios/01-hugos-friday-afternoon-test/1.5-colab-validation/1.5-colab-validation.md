## Overview

**Page Purpose:** Convert the adaptation run into visible, credible proof that Hugo can evaluate and share before leaving the lab.

**User Situation:** Hugo has completed or checkpointed the adaptation loop. He needs to see whether the result is physically plausible enough to show a colleague.

**Main User Goal:** Generate a triptych visualization, read the validation summary, understand time saved, and share the notebook URL.

**Success Criteria:** Hugo sees an animation/GIF with `Imagination`, `Real`, and `Difference` panels, reads an explicit time-to-science callout, and shares the notebook with a colleague.

**Entry Points:**
- `Continue to validation results` from `1.4-colab-adaptation-loop`.
- Direct anchor link to validation results.
- Recovered checkpoint from a previous run.

**Exit Points:**
- Primary: share notebook URL.
- Secondary: export validation artifacts.
- Tertiary: return to debug hooks or adaptation loop if validation fails.

---

## Layout Structure

The validation page should feel like a scientific result card inside the notebook: render the evidence, explain what passed, quantify the time saved, and make sharing low-friction.

```text
+----------------------------------------------------------+
| Validation Header                                        |
| result framing and artifact promise                      |
+----------------------------------------------------------+
| Visualization Cell                                       |
| adapter.visualize(...)                                   |
+----------------------------------------------------------+
| Triptych Result Viewer                                   |
| Imagination / Real / Difference animation or GIF          |
+----------------------------------------------------------+
| Validation Metrics                                       |
| error, smoothness, consistency, confidence                |
+----------------------------------------------------------+
| Time-to-Science Callout                                  |
| run duration vs from-scratch estimate                     |
+----------------------------------------------------------+
| Artifact Export Cell                                     |
| GIF, YAML, run summary                                   |
+----------------------------------------------------------+
| Share Panel                                              |
| copy notebook URL, share with colleague                   |
+----------------------------------------------------------+
| Completion / Recovery Gate                               |
| finish scenario or return to debug/adaptation             |
+----------------------------------------------------------+
```

## Spacing

**Scale:** WDS local tokens: `zero`, `sm`, `md`, `lg`, `xl`, `2xl`, `3xl`.

| Property | Token |
|----------|-------|
| Notebook section padding | Colab markdown default |
| Section gap | `lg` default |
| Visualization panel gap | `sm` |
| Metrics row gap | `sm` |
| Completion boundary | `xl` |

#### <-> `colab-validation-v-space-sm` - header and visualization stay connected

The result framing should flow directly into rendering the artifact.

#### <-> `colab-validation-v-space-sm-viewer` - visualization cell and viewer stay coupled

The viewer is the immediate output of `adapter.visualize()`.

#### <-> `colab-validation-v-space-md-metrics` - viewer to metrics

Metrics explain the visual result and need a readable boundary.

#### <-> `colab-validation-v-space-lg-time` - metrics to time-to-science

The time saved callout is the business/emotional payoff, separate from scientific metrics.

#### <-> `colab-validation-v-space-md-export` - callout to artifact export

Export is a practical continuation of the proof.

#### <-> `colab-validation-v-space-lg-share` - export to share panel

Sharing is the final behavioral goal and should stand out.

**Responsive Rules:**
- Triptych viewer stacks vertically on narrow screens and remains three columns on desktop.
- Difference panel label must remain visible even if animation scales down.
- Metric values must not overflow; long paths use wrapping or horizontal scroll.
- Status must use labels `[PASS]`, `[WARN]`, `[FAIL]`, not color alone.
- No viewport-based font scaling.

## Page Interactions

| Object ID | Interaction |
|-----------|-------------|
| `colab-validation-visualize-code-cell` | User runs the cell. Renderer creates synchronized Imagination, Real, and Difference panels. |
| `colab-validation-triptych-imagination-panel` | Plays or displays model-imagined rollout. |

| `colab-validation-export-code-cell` | User runs the cell. Exports GIF and validation summary artifacts. |
| `colab-validation-share-copy-url-button` | Copies the current notebook or stable share URL. Shows `Copied` for 2 seconds. |

**Global Interaction Rules:**
- Visual output must be inspectable without hidden dependencies.
- The time-to-science callout must use actual run metadata when available.
- Sharing must warn users before exposing edited notebooks or private data.
- If visualization fails, recovery should point to the latest checkpoint and debug hooks.

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page opens after adaptation loop. | Header, visualize cell, empty viewer, export/share sections are visible but not complete. | Run visualize cell. |
| Rendering | `adapter.visualize(...)` is running. | Cell spinner and `Rendering validation triptych...` status. | Wait. |
| Render success | Triptych output is generated. | Three labeled panels appear: Imagination, Real, Difference. | Inspect metrics. |
| Render warning | Output exists but metric confidence is low. | Viewer appears with `[WARN]` verdict and debug link. | Inspect debug hooks or continue with caveat. |
| Render failure | Renderer fails or missing checkpoint/data. | Error callout names missing artifact or invalid state. | Return to adaptation loop or checkpoint recovery. |
| Metrics pass | Validation values meet quickstart thresholds. | `[PASS]` verdict callout. | Export artifacts and share. |
| Metrics warn | Values are plausible but degraded. | `[WARN]` verdict with exact metric reason. | Share with caveat or debug. |
| Metrics fail | Values fail validation thresholds. | `[FAIL]` verdict and debug link. | Return to debug hooks/adaptation. |
| Export complete | Artifacts export successfully. | File list shows GIF, YAML, summary, checkpoint path. | Copy notebook URL. |
| Share copied | Notebook URL is copied. | Button label changes to `Copied` for 2 seconds. | Paste into Slack or message. |
| Scenario complete | Validation artifact and share action are available. | Completion callout appears. | End scenario or continue deeper docs. |

### Component States

| Object ID | States |
|-----------|--------|
| `colab-validation-visualize-code-cell` | Not run, rendering, success, warning, failed. |
| `colab-validation-triptych` | Empty, loading, rendered, partial, failed. |
| `colab-validation-metrics-summary-table` | Empty, pass, warn, fail. |
| `colab-validation-time-saved-metric` | Estimated, measured, unavailable. |
| `colab-validation-export-code-cell` | Not run, exporting, complete, failed. |
| `colab-validation-share-copy-url-button` | Default, copied, clipboard-error. |
| `colab-validation-completion-success-callout` | Hidden, visible. |

## Validation & Errors

| Field / Target | Rule | Timing | Error / Recovery |
|----------------|------|--------|------------------|
| Visualization source | Must have a completed run or valid checkpoint. | `adapter.visualize(...)`. | `No completed run or checkpoint found. Return to adaptation loop.` |
| Validation split | Must contain compatible observation/action shapes. | Visualization cell. | `Validation split mismatch. Regenerate trajectories or rerun configuration summary.` |
| Triptych panels | Must include Imagination, Real, and Difference. | Render output. | `Validation renderer did not produce all required panels.` |
| GIF/animation export | Must produce a viewable artifact. | Export cell. | `Triptych export failed. Check output path and renderer logs.` |
| Time-to-science metric | Must use run metadata or clearly state estimate. | Time callout render. | `Run duration unavailable. Display estimated timing with caveat.` |
| Baseline comparison | Must be sourced, caveated, or softened. | Publication review. | `Baseline claim needs evidence or softer wording.` |
| Export artifact list | Must include GIF, summary, YAML, and checkpoint path when available. | Export cell. | `Artifact list incomplete. Rerun export after successful validation.` |
| Clipboard share | Must provide fallback if URL copy fails. | Share button click. | `Select and copy the notebook URL manually.` |
| Private data warning | Must appear before sharing edited/private notebooks. | Share panel. | `Save a clean copy before sharing private data.` |

## Conditional Sections

| Condition | Include |
|-----------|---------|
| Public notebook | Keep validation data synthetic or explicitly safe to share. |
| Visual result | Render labels must remain visible in exported GIF/animation. |
| Metrics warning/failure | Provide return link to debug hooks, not generic failure text. |
| Sharing | Warn about edited cells and private data before sharing. |
| Accessibility | Triptych panels need text labels and metric table alternatives. |
| Open questions | Confirm final renderer API, metric thresholds, export schema, and share URL behavior. |
