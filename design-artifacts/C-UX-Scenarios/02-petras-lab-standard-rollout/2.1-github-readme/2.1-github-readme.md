## Overview

**Page Purpose:** Pass Petra's 60-second institutional credibility filter — MIT license, scientific authority (ArXiv), active maintenance, and sustainability signals — so she can stake her professional reputation on recommending PhysLink to Professor Paula.

**User Situation:** Petra is at her lab desk on Monday morning following Hugo's success on Friday. She needs to answer a binary question before the weekly meeting with Paula: "Can I stake my reputation on this tool?"

**Main User Goal:** Confirm in under 60 seconds that PhysLink is scientifically credible, institutionally safe, and actively maintained. Then follow the CHANGELOG to assess maintenance cadence.

**Success Criteria:** Petra locates the MIT License badge, finds the ArXiv citation link, notes recent commit activity, and clicks the Changelog link to move to `2.2-changelog`. She exits the README with enough confidence to begin drafting her recommendation.

**Entry Points:**
- Hugo's Slack message sharing the link after his Friday test.
- Hugo's direct referral (internal lab communication).

**Exit Points:**
- Primary: `Changelog` link (maintenance history validation).
- Secondary: `Read the paper` / ArXiv link (scientific authority validation).
- Tertiary: `Lab Adoption Guide` (content for her recommendation memo).
- Quaternary: `Open in Colab` (she may check the live notebook as evidence for Paula).

## Petra's Scan Priority

Petra reads the same page as Hugo, but in a different order and with different stopping conditions. She is not looking for a fast technical win — she is performing a due-diligence checklist.

| Priority | Object | Signal she's reading | Pass condition |
|----------|--------|----------------------|----------------|
| 1 | `readme-repository-license-badge` | Institutional safety — can the lab adopt this freely? | License is MIT or equivalent permissive |
| 2 | `readme-repository-arxiv-badge` | Scientific authority — is this peer-reviewed or positioned for it? | ArXiv link present and reachable |
| 3 | `readme-repository-maintenance-badge` *(new)* | Freshness — is this actively maintained? | Latest release / last commit within ≤ 6 months |
| 4 | `readme-repository-ci-badge` | Quality hygiene — does the CI pass? | Green badge |
| 5 | `readme-hero-primary-headline` | Problem framing — does this solve a real scientific problem? | Headline names a research pain, not just a technical feature |
| 6 | `readme-action-read-paper-link` | Academic depth — is there a proper paper? | ArXiv link visible and labeled `Read the paper` |
| 7 | `readme-links-changelog-link` | Maintenance cadence — is the project maintained, not just launched? | CHANGELOG link present and prominent |
| 8 | `readme-links-lab-adoption-link` | Institutional path — is there documentation for labs, not just developers? | Lab Adoption Guide link present |
| 9 | `readme-footer-pr-template-link` | Community signal — is there an active contributor community? | Contribution guide link present |

**Stopping conditions:**
- **GO**: All 9 signals found → clicks `Changelog` to move to `2.2`.
- **NO-GO**: License is not permissive, OR ArXiv link is absent, OR no maintenance activity in 6+ months → Petra closes the tab; PhysLink is not recommended.

## Layout Structure

The page layout is shared with `1.1`. The README must serve both personas simultaneously. For Petra, the badge row (Repository Header) carries the most weight — she decides within the first viewport whether to continue.

Design tension: The current layout front-loads code (Hugo's path). For Petra, the code example is secondary — she reads it as evidence of quality but does not need to understand it. The Fast Action Bar must surface an institutional path alongside the Colab CTA.

```text
+----------------------------------------------------------+
| Repository Header          [PETRA: ★★★ PRIMARY]          |
| repo title · CI · License · arXiv · Maintenance badges   |
+----------------------------------------------------------+
| Trust Filter Hero          [PETRA: ★★ secondary]         |
| headline + short problem hook                            |
+----------------------------------------------------------+
| Foundational Code Example  [PETRA: ★ low]                |
| Python mirror-code + copy action                         |
+----------------------------------------------------------+
| Result Proof               [PETRA: ★ low]                |
| terminal output + time-saved callout                     |
+----------------------------------------------------------+
| Fast Action Bar            [PETRA: ★★★ PRIMARY]          |
| Open in Colab · Read the paper · → Evaluate for your lab |
+----------------------------------------------------------+
| Installation & Quickstart  [PETRA: ★ low]                |
| pip install + physlink doctor                            |
+----------------------------------------------------------+
| Deep Trust Links           [PETRA: ★★★ PRIMARY]          |
| Changelog · Lab Adoption Guide · Domain Scientists       |
+----------------------------------------------------------+
| Footer / Contribution      [PETRA: ★★ secondary]         |
| PR template · issues · license · contributor count       |
+----------------------------------------------------------+
```

## Spacing

**Scale:** WDS local tokens: `zero`, `sm`, `md`, `lg`, `xl`, `2xl`, `3xl`.

| Property | Token |
|----------|-------|
| Page padding (horizontal) | GitHub README default content gutter |
| Section gap | `lg` default |
| Element gap within sections | `md` default |
| Component gap within groups | `sm` default |

#### ↕ `readme-v-space-md` — repository header and hero remain close

The badge row and the headline form one continuous credibility surface. Petra reads them as a unit.

#### ↕ `readme-v-space-lg` — hero breathes before code example

The code example must feel like a supporting detail, not the main pitch. Breathing room reinforces this hierarchy.

#### ↕ `readme-v-space-sm` — result belongs directly to code

Terminal output is the direct consequence of the code. Same as 1.1.

#### ↕ `readme-v-space-md-action` — action bar follows proof

The action bar appears after the evidence. Same as 1.1.

#### ↕ `readme-v-space-xl` — installation is secondary content

Clear boundary before the quickstart section, which Petra reads last if at all.

#### ↕ `readme-v-space-lg-trust` — deep trust links are elevated for Petra

For Hugo these were a compact secondary block. For Petra they are her primary navigation target. The spacing should signal relevance without competing with the action bar above.

#### ↕ `readme-v-space-xl-footer` — footer is a final signal, not the destination

Contribution links confirm community health but appear only after Petra has already formed her GO/NO-GO decision.

## Validation & Errors

This README has no user input form. Validation is pre-release / CI documentation quality checks plus runtime link fallback behavior.

| Field / Target | Rule | Timing | Error / Recovery |
|----------------|------|--------|------------------|
| MIT License badge | Must display MIT or compatible permissive license. | Pre-release check. | Petra exits if license is absent or restrictive. |
| arXiv badge and link | Must point to a reachable public arXiv page. | Pre-release / CI docs check. | `Paper link is unavailable` — Petra notes as credibility risk. |
