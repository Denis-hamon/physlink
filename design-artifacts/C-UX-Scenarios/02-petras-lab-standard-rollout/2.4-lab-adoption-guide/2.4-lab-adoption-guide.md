## Overview

**Page Purpose:** Give Petra the institutional content she needs to build her recommendation memo for Paula. She is not reading this guide to learn how to use PhysLink — she is reading it to confirm that a structured path exists for a lab to adopt it collectively.

**User Situation:** Petra has confirmed: MIT license ✓, CHANGELOG active ✓, arXiv paper ✓. She now needs to answer Paula's likely question: *"What does this mean practically for our lab?"* This page must give her copy-paste answers.

**Main User Goal:** Find and copy 3–5 concrete features that make PhysLink a lab-level (not individual-level) tool: shared infrastructure, reproducibility standards, multi-user workflows, onboarding path.

**Success Criteria:** Petra copies the Lab Adoption Guide URL into her memo. She extracts at least 2 features to cite for Paula (e.g., `AdaptationJobs` for queued multi-user runs, buffer sharing for reproducible checkpoints). She exits to the GitHub Templates page (2.5) for the final sustainability check.

**Entry Points:**
- Click on `Lab Adoption Guide` from `readme-links-lab-adoption-link` (2.1 Deep Trust Links).
- Click on `Evaluate for your lab →` from `readme-action-lab-path-link` (2.1 Fast Action Bar).
- Direct URL from arXiv paper body reference.

**Exit Points:**
- Primary: GitHub Templates (2.5) — Petra's final sustainability check.
- Secondary: Copy the page URL into her memo (completed in-place — no navigation required).
- Tertiary: Navigate to a specific sub-section linked from Paula's open question.

## Spacing

**Scale:** WDS local tokens: `zero`, `sm`, `md`, `lg`, `xl`, `2xl`, `3xl`.

Documentation site uses a standard content width with a fixed left sidebar.

| Property | Token |
|----------|-------|
| Content max-width | Documentation theme default (~800px) |
| Section gap | `xl` — each major section is a distinct chapter |
| Element gap within sections | `md` |
| Code block margin | `lg` above and below |

#### ↕ `lab-guide-v-space-xl` — major sections are discrete chapters

Petra navigates by section heading, not by reading continuously. Clear vertical separation lets her jump-to-section without losing context.

#### ↕ `lab-guide-v-space-md` — code blocks breathe within their section

Code examples (AdaptationJobs API, buffer sharing) need space above and below to read as distinct reference blocks, not inline prose interruptions.
