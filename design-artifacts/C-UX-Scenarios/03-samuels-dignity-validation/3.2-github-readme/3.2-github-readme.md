## Overview

**Page Purpose:** Give Samuel a "For Domain Scientists" entry point visible in under 10 seconds. He arrives already converted by the arXiv paper — he does not need to be convinced. He needs to confirm there is a path built for people like him.

**User Situation:** Samuel has just read the PhysLink arXiv paper and clicked the GitHub link. He is a domain scientist (CFD), not an ML engineer. He fears being ignored by a tool that treats physics as an afterthought. His confidence is fragile: he is looking for a single signal that the project acknowledges his kind of user.

**Main User Goal:** Find the "For Domain Scientists" link in under 10 seconds.

**Success Criteria:** Samuel locates `readme-links-domain-scientists-link` and clicks it. Time on page < 10 seconds if the link is visible in the first viewport or easily scannable.

**Entry Points:**
- `arxiv-paper-github-link` from the arXiv paper body (3.1).

**Exit Points:**
- Primary: `readme-links-domain-scientists-link` → Getting Started for Domain Scientists (3.3).
- NO-GO: Samuel scans the README and finds no domain scientist entry point in 10 seconds → he concludes "this is for ML engineers, not me" and closes the tab.

## Samuel's Scan Pattern

Samuel's time budget on this page is the shortest of all three scenarios.

| Step | Object | What he's doing | Time |
|------|--------|-----------------|------|
| 1 | `readme-repository-header` | Confirms he is on the right repo (he already knows the name from the paper). | 1 s |
| 2 | `readme-hero-primary-headline` | Reads the headline to confirm "adaptation" language. | 2 s |
| 3 | `readme-links` section | **Scans link list for "Domain Scientists" entry.** This is his single target. | 4 s |
| 4 | `readme-links-domain-scientists-link` | Finds and clicks it. | 1 s |

**Total: < 10 seconds if the "For Domain Scientists" link is in the Deep Trust Links section.**

**Critical design constraint:** If the "For Domain Scientists" link is below the fold (requires scrolling past the code example and result proof), Samuel may not find it within his 10-second budget. This reinforces the redesign proposal from 2.1: the Deep Trust Links section needs visual elevation, or the `readme-action-lab-path-link` CTA should be extended with a "For Domain Scientists" path in the Fast Action Bar.

---

## Design Tension: Discoverability for Samuel

The current README layout (from 1.1) places the "For Domain Scientists" link in the **Deep Trust Links** section — secondary priority, below the code example, result proof, action bar, and installation section. For Hugo (90-second budget, looking for code first) this is appropriate. For Samuel (10-second budget, looking for a domain entry point), this position may be too deep.

**Options (open question):**

| Option | Trade-off |
|--------|-----------|
| Keep link in Deep Trust Links | Works for Hugo; risky for Samuel's 10s budget unless the section is above the fold on a 1440px monitor |
| Add "For Domain Scientists" to the Fast Action Bar | Serves Samuel immediately; adds a third path to the action bar (Colab / Paper / Domain Scientists) |
| Add a "Who is this for?" anchor section above Deep Trust Links | A one-row grid of 3 user types (Researcher / Lab / Domain Scientist) with direct links — maximum discoverability |
